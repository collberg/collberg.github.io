Self Modify

free web templates

The goal of these transformations is to maske functions self-modifying. Currently, only indirect branches are transformed. This transformation is therefore particularly useful after transformations that introduce indirect branches, such as virtualization with a direct or indirect dispatch. Here's an example, transforming a function add that only does x++:

> tigress  --Environment=x86_64:Darwin:Clang:5.1 \
         --Transform=Virtualize\
            --Functions=add \
            --VirtualizeDispatch=direct \
         --Transform=SelfModify \
            --Functions=add \
   inc.c --out=obf.c

> gcc -segprot __TEXT rwx rwx obf.c -o obf.exe   # Darwin

> gcc --static -Wl,--omagic obf.c -o obf.exe     # Linux

Note that we need to compile the generated code in such a way that the text segment pages are writable. See here for alternatives.

Each indirect branch turns into the following code, where the byte sequence corresponds to the X86 8-byte direct jump:

  addr = (unsigned long *)((unsigned long )(&& Lab) + 6);
  *addr = address-to-jump-to;
  Lab: asm volatile(".byte 0xff, 0x25, 00, 00, 00, 00, 00, 00, 00, 00, 00, 00, 00, 00":);

OptionArgumentsDescription
--Transform SelfModify Transform the function by adding code that modifies itself.
--SelfModifyKinds indirectBranch Instructions to self-modify. Currently only indirect branches can be transformed. Default=indirectBranch.
  • indirectBranch = Transform indirect branches
